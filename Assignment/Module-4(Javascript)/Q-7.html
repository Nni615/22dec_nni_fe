7) What is the difference between Viewstate and SessionState?



->An user has a STATE it could be View State and Session State.
->State Management:-
The Management of the state becomes more difficult as an application grows.
Web is stateless which means every time a specific webpage is requested, and it is recreated each time and posted to the server.
->Viewstate and SessionState are used for client-side state management and server-side state management.


Differences...

ViewState SessionState
It is an Client-side State Management. It is an Server-side State Management.
Data will be accessed within the same webpage. Data will be accessed from other webpages of the website.
When new pages are loaded, ViewState values
are lost/cleared. 		SessionState will be cleared when it is cleared by the programmer, by the user,
and when Timeout occurs.
Data is stored in an hidden field of the same webpage. Data is stored in the sessions and cookies.
